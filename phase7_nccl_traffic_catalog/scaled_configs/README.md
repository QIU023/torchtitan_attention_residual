# Scaled IXIA configs — datacenter projections of 8-GPU NCCL traces

8-GPU AttnRes-Kimi-Linear NCCL traces (phase 5/6/9/10/11) projected onto
published large-LLM 5D parallelism plans. Generated by
`phase7_nccl_traffic_catalog/scale_ixia_to_dc.py`.

The source `ixia_config.json` files (8-GPU `tier_b` traces from torchtitan
runs on this project's box) are valid for **algorithm-level** fabric
profiling — collective op counts, message sizes, axis topology — but
their absolute throughput is bounded by 8x RTX 5090 PCIe Gen5
(~111 GB/s aggregate, ~14 GB/s per rank). To project onto datacenter
scale we need **two simultaneous scalings**:

1. **Endpoint multiplication** — 8 ranks → 2,048 / 10,240 / 16,384 ranks
   following the target plan's mesh (`fsdp × pp × tp × ep × cp`).
2. **Per-call byte rescaling** — 436M params / 1168 hidden / 16 layers /
   256 seq → target's params / hidden / layers / seq, applied
   per-axis (FSDP scales by params×inverse-shard-count, TP scales by
   hidden, PP scales by hidden×seq, EP scales by hidden×seq×top-k).

The script also adjusts inter-frame gap to saturate a configurable link
bandwidth (default 400 Gbps NDR-class) at ~80% line rate using jumbo
9000-byte frames.

## Target topologies

All three are encoded in `phase7_nccl_traffic_catalog/scale_ixia_to_dc.py` `TARGET_PRESETS`:

| Preset | World | Mesh (FSDP × PP × TP × EP × CP) | Model | Reference |
|---|---|---|---|---|
| `llama3-405b-16k` | 16,384 | 128 × 16 × 8 × 1 × 1 | 405B dense | Llama 3 paper, Meta AI 2024 (arXiv:2407.21783) |
| `deepseek-v3-671b-2k` | 2,048 | 16 × 16 × 1 × 8 × 1 | 671B / 37B-active MoE | DeepSeek-V3 technical report, 2024 (arXiv:2412.19437) |
| `kimi-k2-style-10k` | 10,240 | 20 × 16 × 4 × 8 × 1 | 1T / 50B-active MoE | Synthesized — DeepSeek-V3 + Kimi K2 public hints |

## How to use these in IxNetwork (Windows GUI)

The schema is `ixnetwork-traffic-config-v1`. The intended ingest path:

1. **Endpoints**: each `topology.endpoints[i]` is one IxNetwork port with
   synthetic IPv4 + MAC. For target world > available chassis ports,
   use IxNetwork's "Endpoint Set Multiplier" or virtual ports.
2. **Traffic items**: each `trafficItems[i]` is one IxNetwork
   `trafficItem` of type `L2L3` with metadata describing
   `axis_guess` (fsdp / pp / tp / ep / cp), `target_axis_ranks`
   (number of parallel rings of this collective at target scale),
   and `bytes_total` (cumulative bytes for the targeted step window).
3. **Frame rate**: `interFrameGap_ns` is set to ~80% line rate at the
   chosen `--link-bandwidth-gbps` with jumbo 9000-byte frames.

Workflow:

```
File → Import Configuration → ixia_config.json
Endpoint Sets → Multiplier = N (per replica)
Stream Properties → Rate = "Line Rate %" 80%
Run Traffic
```

## Caveats

* This is a **pure replay** of an 8-GPU mesh's collective sequence
  with bytes rescaled; the real Llama 3 405B training plan issues a
  qualitatively different schedule (deeper PP, more microbatches per
  step, async DP-replicate barriers) than our 8-GPU box produces.
  Use these configs as **fabric volume** sizing input, not as a
  faithful replay of paper training behavior.
* Per-axis byte scaling uses simplified ratios (params, hidden, seq,
  top-k). Real production includes activation recomputation, gradient
  bucketing, optimizer-state sharding, communication compression —
  none of which is modeled here. Treat absolute Gbps numbers as
  ±2× envelopes.
* `kimi-k2-style-10k` is synthesized; not a paper-faithful plan.
  Marked as such in the output `target.ref` metadata.

## Regenerating

```bash
python phase7_nccl_traffic_catalog/scale_ixia_to_dc.py \
    --source phase5_vlm_multimodal_sft/runs/<config>/tier_b_trace/ixia_config.json \
    --preset llama3-405b-16k \
    --source-steps 5000 --target-steps 50 \
    --target-step-time-sec 5.0 \
    --link-bandwidth-gbps 400 \
    --out phase7_nccl_traffic_catalog/scaled_configs/<name>.json
```
